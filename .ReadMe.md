# ISDN System — Integrated Supply & Distribution Network

A multi-role web-based supply chain and order management system built with **PHP, MySQLi, and Bootstrap**. It covers the full order lifecycle — from customer purchase through inventory management, inter-branch stock transfers, delivery tracking, returns, and financial reporting — across five distinct user roles.

## User Roles

| Role | Dashboard | Key Responsibilities |
|---|---|---|
| **Admin** | `admin_dashboard.php` | Manage users, distribution centres, assign logistics staff |
| **HO Manager** | `ho_dashboard.php` | Head-office oversight, purchase orders, revenue & performance reports |
| **RDC Staff** | `rdc_dashboard.php` | Regional distribution centre — inventory, stock transfers, sales reports |
| **Logistics Staff** | `logistics_dashboard.php` | Manage and update deliveries, live location updates |
| **Customer** | `customer_dashboard.php` | Browse products, cart, place orders, track deliveries, request returns |

## Features

- **Products & Inventory** — create/edit products, view stock levels, search, inter-branch stock transfers between RDCs
- **Orders** — customers add to cart, place orders, make payments via the payment gateway module
- **Purchase Orders** — HO creates POs to restock from suppliers; RDC staff view and fulfil them
- **Invoices** — auto-generated PDF invoices (FPDF) emailed to customers via PHPMailer/SMTP
- **Delivery Tracking** — logistics staff update delivery status and live location via REST-style API (`api_update_location.php`); customers track shipments
- **Returns** — customers request returns; RDC staff process and approve/reject
- **Reports** — stock reports, sales reports, revenue reports, RDC weekly summaries, due-payment reports, performance reports, RDC stock analysis
- **Role-based access control** — every page enforces role checks via `includes/auth_check.php`

## Tech Stack

- **Backend:** PHP (procedural), MySQLi
- **Database:** MySQL
- **PDF Generation:** FPDF (bundled in `includes/libs/`)
- **Email:** PHPMailer (bundled in `includes/libs/phpmailer/`)
- **Frontend:** Bootstrap, HTML/CSS, JavaScript
- **Local Dev Server:** XAMPP (Apache + MySQL)

## Prerequisites

- **XAMPP** (or any Apache + PHP + MySQL stack)
- PHP 7.4+
- MySQL 5.7+ / MariaDB

## Setup

1. **Clone the repository**
   ```
   git clone https://github.com/<your-username>/ISDN-System.git
   ```

2. **Copy into your XAMPP htdocs folder**
   ```
   Move the ISDN-System folder to: C:\xampp\htdocs\ISDN-System
   ```

3. **Import the database**
   - Start XAMPP and open [http://localhost/phpmyadmin](http://localhost/phpmyadmin)
   - Create a new database called `isdn_db`
   - Import `database/isdn_db.sql`, then `database/stock_transfer_schema.sql`, then `database/updates.sql` (in that order)

4. **Create the database config file**
   The file `config/db.php` is excluded from this repository for security. Create it manually:
   ```php
   <?php
   define('DB_HOST', 'localhost');
   define('DB_USER', 'root');
   define('DB_PASS', '');       // Your MySQL password
   define('DB_NAME', 'isdn_db');

   $conn = mysqli_connect(DB_HOST, DB_USER, DB_PASS, DB_NAME);
   if (!$conn) {
       die("Database connection failed: " . mysqli_connect_error());
   }
   mysqli_set_charset($conn, 'utf8mb4');
   ```

5. **Configure email (for invoice sending)**
   In `includes/invoice_service.php`, update the PHPMailer credentials with your own SMTP details:
   ```php
   $mail->Username = 'your_email@gmail.com';
   $mail->Password = 'your_app_password';   // Use a Gmail App Password, not your real password
   ```

6. **Run the application**
   Visit [http://localhost/ISDN-System](http://localhost/ISDN-System) in your browser.

7. **Create your first admin account**
   Register through the app and then manually update the `role` column in the `users` table via phpMyAdmin to `admin` for your first account. After that, the admin can create other users with correct roles from the dashboard.

## Security Notes

- `config/db.php` is listed in `.gitignore` — **never commit real database credentials** to a public repository.
- If you ever had your SMTP password in `invoice_service.php` committed to a public repo, revoke and regenerate that Gmail App Password immediately.
- The `temp/invoices/` folder holds generated PDFs temporarily and is excluded from version control.
